# Author-as-editor with pre-committed decision rules

The Editor control headline axis is operationalised by the author playing the editor role when demonstrating each Pattern. **Selection bias is real**; the methodology section acknowledges this explicitly. To reduce it without giving up the convenience of author-as-evaluator, the **accept / edit / reject decision rules** for each Pattern are defined in the telemetry schema **before the Pattern is implemented and before any AI output is observed**. Pre-committing prevents post-hoc rationalisation; subsequent edits to the rules are themselves an audit signal.

Synthetic personas were rejected: they would not produce real measurements, only usability illustrations. Real Vejle Kommune editors are pursued as a stretch via the digitalisering outreach from ADR-0004. If the outreach lands, author-as-editor numbers become "preliminary validation, subsequently corroborated by N real-editor sessions" — a meaningful upgrade. If it doesn't, the thesis is intact at the author-as-editor baseline.

Thesis introduction scopes the catalogue with the exact language: *"This catalogue scopes to Editorial-path and Design-time-path Patterns — AI that operates during content creation, review, and publication. Runtime-path AI (personalisation engines, AI-ranked search results, recommendation systems) is excluded; it has its own established literature and different architectural constraints outside this thesis's scope."* This pre-empts the examiner question about omissions and frames the scope as a deliberate analytical choice.
